# 关于仅使用浏览器远程控制家中电脑的方案

我们学校最近新出了一个BYOD计划，即Bring Your Own Devices，能够让我们在学校使用ipad来学习。但ipad不能下载软件，本质上就是一个昂贵的浏览器。于是我便用一台云服务器，打造了这个方案，能使我在学校远程控制家中的电脑。
（ps：我在之后才发现了一套可能更简单的方案，详情见https://www.bilibili.com/video/BV1937NzwE2Q/和https://www.bilibili.com/video/BV1FV4xz9EPt 你可以选一个适合你的方案）

## 1.概览
本方案可令ipad访问部署在云服务器上的tomcat上的guacamole，通过ssh隧道使用vnc连接远程连接到家中的电脑，进而实现WOL（wake on lan），文件传输等功能。
此方案完全基于Web浏览器（如iPad自带的Safari）运行，无需在iPad上安装任何App，完美规避限制。
此外，整个访问链路被分解为两段并分别进行高强度加密。iPad到云服务器之间通过HTTPS (TLS/SSL)加密，云服务器到家庭电脑之间通过SSH隧道加密，确保了数据在传输过程中的机密性和完整性。
所有远程连接配置都集中在云服务器的Guacamole网关上，方便管理，并且未来可以轻松扩展以支持更多设备或用户。
该方案还有卓越的兼容性，无论电脑是Windows、macOS还是Linux操作系统，均能通过协议连接。

## 2.流程
1. 建立隧道： 家庭电脑启动后，会自动运行一个SSH客户端程序，主动连接到云服务器的SSH服务端口（通常是22），并建立一个SSH反向隧道。此隧道的作用是，将云服务器上的一个本地端口（例如127.0.0.1:33890）的流量，安全地转发到家庭电脑的远程桌面服务端口。
2. 用户访问： 用户在公司的iPad上打开Safari浏览器，输入一个指向你云服务器的域名（例如 https://remote.yourdomain.com）。
3. Web网关处理： 云服务器上的Web服务器（如Nginx）接收到HTTPS请求，将其反向代理到后端运行的Apache Guacamole服务。
4. 身份验证： Guacamole提供一个登录界面。用户输入预设的用户名和密码进行身份验证。
5. 建立远程会话： 验证通过后，用户在Guacamole界面中点击代表家庭电脑的连接。Guacamole的后端进程 (guacd) 会根据配置，尝试连接到云服务器自身的本地端口 127.0.0.1:33890。
6. 隧道转发： SSH服务监听到这个连接请求，立即通过已经建立好的反向隧道，将数据包加密后转发给家庭电脑。
7. 会话开始： 家庭电脑的远程桌面服务（RDP或VNC）收到连接请求，开始会话。屏幕画面、鼠标和键盘数据通过“家庭电脑 -> SSH隧道 -> 云服务器 -> Guacamole -> HTTPS -> iPad浏览器”的路径进行双向传输。

## 3.方案中的组件
### Apache Guacamole：
Apache Guacamole是一个开源的、无需客户端的远程桌面网关。它通过将RDP、VNC、SSH等标准协议“翻译”成HTML5，让用户可以通过任何现代Web浏览器进行远程访问。

### ssh反向隧道：
 SSH (Secure Shell) 协议不仅用于远程登录，其端口转发功能也极为强大。反向隧道（Remote Port Forwarding）允许位于NAT后的机器主动向公网服务器发起连接，并请求服务器监听一个端口，然后将该端口的流量转发回这台机器。

### Nginx + Let's Encrypt
 Nginx是一款高性能的Web服务器和反向代理服务器。Let's Encrypt是一个免费、自动化、开放的证书颁发机构（CA），可以轻松获取和续订SSL/TLS证书。

## 4.实操
### 4.1 准备工作

- **一台云服务器**：任意主流云服务商（AWS、阿里云、腾讯云等），操作系统推荐 Ubuntu 22.04/24.04 LTS。至少 1 核 CPU、2GB 内存、20GB 磁盘。需开放 22（SSH）、80（HTTP）、443（HTTPS）端口（安全组/防火墙）。
- **一个域名**（可选，强烈推荐）：用于指向云服务器公网 IP，例如 remote.yourdomain.com。如无域名，可使用 IP 地址 + 自签名证书，但浏览器会提示不安全。
- **家庭电脑**：
  - **Windows**：Windows 10/11 专业版/企业版/教育版（内置 RDP 服务端）；家庭版需安装第三方 VNC Server（如 TigerVNC）。
  - **macOS**：系统内置屏幕共享（VNC），需在“共享”中开启。
  - **Linux**：需安装并配置 VNC Server。本文以 Arch Linux + Hyprland (Wayland) 为例，使用 wayvnc。
- **iPad**：能正常上网。

### 4.2 云服务器基础配置
#### 4.2.1 登录云服务器并更新系统
```bash
ssh root@你的服务器IP
sudo apt update && sudo apt upgrade -y
```

#### 4.2.2 安装 Docker 和 Docker Compose
```bash
sudo apt install -y docker.io docker-compose
sudo systemctl enable --now docker
# 将当前用户加入 docker 组，避免每次使用 sudo
sudo usermod -aG docker $USER
# 重新登录或执行 newgrp docker 使组生效
```

验证安装：
```bash
docker --version
docker-compose --version
```

#### 4.2.3 安装 Nginx
```bash
sudo apt install -y nginx
```

### 4.3 使用 Docker Compose 部署 Guacamole 服务

我们将通过 Docker Compose 一键部署 Guacamole 及其依赖（guacd、PostgreSQL），并自动完成数据库初始化。

#### 4.3.1 创建项目目录
```bash
mkdir ~/guacamole && cd ~/guacamole
```

#### 4.3.2 编写 docker-compose.yml

创建 `docker-compose.yml` 文件：
```bash
nano docker-compose.yml
```

粘贴以下内容（**请将 YOUR_STRONG_PASSWORD 替换为你的强密码**）：
```yaml
version: '3'

services:
  guacd:
    image: guacamole/guacd:latest
    container_name: guacd
    restart: unless-stopped
    networks:
      - guacamole-net

  postgres:
    image: postgres:13
    container_name: postgres
    restart: unless-stopped
    environment:
      POSTGRES_DB: guacamole_db
      POSTGRES_USER: guacamole_user
      POSTGRES_PASSWORD: YOUR_STRONG_PASSWORD
    volumes:
      - ./postgres-data:/var/lib/postgresql/data
    networks:
      - guacamole-net

  init-db:
    image: guacamole/guacamole:latest
    container_name: init-db
    command: ["/opt/guacamole/bin/initdb.sh", "--postgresql"]
    environment:
      POSTGRES_HOSTNAME: postgres
      POSTGRES_DATABASE: guacamole_db
      POSTGRES_USER: guacamole_user
      POSTGRES_PASSWORD: YOUR_STRONG_PASSWORD
    depends_on:
      - postgres
    networks:
      - guacamole-net
    # 该容器执行完毕后会自动退出

  guacamole:
    image: guacamole/guacamole:latest
    container_name: guacamole
    restart: unless-stopped
    depends_on:
      - guacd
      - init-db
    environment:
      GUACD_HOSTNAME: guacd
      POSTGRES_HOSTNAME: postgres
      POSTGRES_DATABASE: guacamole_db
      POSTGRES_USER: guacamole_user
      POSTGRES_PASSWORD: YOUR_STRONG_PASSWORD
    ports:
      - "8080:8080"   # 将容器 8080 映射到主机 8080，供 Nginx 反向代理
    networks:
      - guacamole-net

networks:
  guacamole-net:
    driver: bridge
```

> **说明**：
> - `init-db` 服务会在 PostgreSQL 容器启动后自动执行数据库初始化脚本，创建 Guacamole 所需的表结构和默认管理员用户（`guacadmin`/`guacadmin`）。
> - Guacamole 容器通过环境变量直接连接数据库，无需手动配置 JDBC 驱动或放置任何扩展 JAR。

#### 4.3.3 启动服务
```bash
docker-compose up -d
```

等待几分钟，所有容器启动后，检查状态：
```bash
docker-compose ps
```

所有服务应为 `Up` 状态。

### 4.4 配置 Nginx 反向代理与 SSL
#### 4.4.1 配置 Nginx（有域名 + Let's Encrypt）

如果你有域名（如 `remote.yourdomain.com`），推荐使用 Let's Encrypt 获取免费可信证书。

1. **安装 Certbot**：
   ```bash
   sudo apt install -y certbot python3-certbot-nginx
   ```

2. **创建初始 Nginx 配置**（用于 Certbot 验证）：
   ```bash
   sudo nano /etc/nginx/sites-available/guacamole
   ```
   写入：
   ```nginx
   server {
       listen 80;
       server_name remote.yourdomain.com;
       location / {
           root /var/www/html;
           index index.html;
       }
   }
   ```

3. **启用配置**：
   ```bash
   sudo ln -s /etc/nginx/sites-available/guacamole /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl reload nginx
   ```

4. **获取 SSL 证书**：
   ```bash
   sudo certbot --nginx -d remote.yourdomain.com
   ```
   按提示完成，Certbot 会自动修改 Nginx 配置并启用 HTTPS。

5. **修改 Nginx 配置以反向代理 Guacamole**：
   Certbot 生成的配置中可能已包含 SSL 部分，只需将 `location /` 修改为：
   ```nginx
   location / {
       proxy_pass http://127.0.0.1:8080/guacamole/;
       proxy_buffering off;
       proxy_http_version 1.1;
       proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
       proxy_set_header Upgrade $http_upgrade;
       proxy_set_header Connection $http_connection;
       proxy_cookie_path /guacamole/ /;
       access_log off;
   }
   ```
   保存后重启 Nginx：
   ```bash
   sudo systemctl restart nginx
   ```

#### 4.4.2 无域名时使用自签名证书

如果你没有域名，可以使用 IP 地址 + 自签名证书（浏览器会提示不安全，但数据仍加密）。

1. **生成自签名证书**（将 `你的服务器IP` 替换为实际 IP）：
   ```bash
   sudo mkdir -p /etc/ssl/private /etc/ssl/certs
   sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
       -keyout /etc/ssl/private/nginx-selfsigned.key \
       -out /etc/ssl/certs/nginx-selfsigned.crt \
       -subj "/CN=你的服务器IP"
   ```

2. **创建 Nginx 配置** `/etc/nginx/sites-available/guacamole`：
   ```nginx
   server {
       listen 80;
       server_name 你的服务器IP;
       return 301 https://$host$request_uri;
   }

   server {
       listen 443 ssl;
       server_name 你的服务器IP;

       ssl_certificate /etc/ssl/certs/nginx-selfsigned.crt;
       ssl_certificate_key /etc/ssl/private/nginx-selfsigned.key;
       ssl_protocols TLSv1.2 TLSv1.3;
       ssl_ciphers HIGH:!aNULL:!MD5;

       location / {
           proxy_pass http://127.0.0.1:8080/guacamole/;
           proxy_buffering off;
           proxy_http_version 1.1;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection $http_connection;
           proxy_cookie_path /guacamole/ /;
           access_log off;
       }
   }
   ```

3. **启用配置并重启 Nginx**：
   ```bash
   sudo ln -s /etc/nginx/sites-available/guacamole /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl restart nginx
   ```

### 4.5 验证 Guacamole 是否可访问

在浏览器中访问 `https://你的域名或IP`，应显示 Guacamole 登录页面。  
默认管理员用户名为 `guacadmin`，密码为 `guacadmin`。**首次登录后请立即修改密码**（右上角用户名 → Settings → Preferences → Change Password）。

### 4.6 阶段三：配置家庭电脑

现在我们需要配置家庭电脑，让它能够被远程访问，并且建立到云服务器的安全隧道。

#### 4.6.1 针对 Windows (10/11 Pro/Enterprise) 系统

- **启用远程桌面 (RDP)**：
  - 打开“设置” -> “系统” -> “远程桌面”。
  - 打开“启用远程桌面”开关。
  - 记下你的电脑用户名。你需要用这个用户名和密码来登录。
  - **注意**：如果是Windows家庭版，此功能不可用。请跳到文末关于VNC的说明。

- **建立SSH反向隧道**：
  - Windows 10/11已内置OpenSSH客户端。打开“PowerShell”或“命令提示符”。
  - 执行以下命令来建立隧道。
    - `remote.yourdomain.com`：你的云服务器域名。
    - `ubuntu`：你登录云服务器的用户名。
    - `33890`：这是云服务器上监听的端口，可以自定义，只要不冲突即可。
    - `localhost:3389`：3389是Windows RDP服务的默认端口。
    ```bash
    ssh -N -R 33890:localhost:3389 ubuntu@remote.yourdomain.com
    ```
    首次连接会提示确认服务器指纹，输入`yes`。然后输入你云服务器用户的登录密码或SSH密钥密码。  
    `-N`参数表示不执行远程命令，只用于端口转发。  
    隧道建立后，这个PowerShell窗口会保持运行，**不要关闭它**。

- **使隧道持久化（推荐）**：
  每次重启电脑后手动运行命令很不方便。我们可以将其设置为一个Windows服务，实现开机自启和断线重连。
  - **使用SSH密钥免密登录**：为了自动化，必须配置SSH密钥。在你的Windows电脑上生成密钥对，并将公钥上传到云服务器的`~/.ssh/authorized_keys`文件中。
  - **使用 nssm (Non-Sucking Service Manager) 创建服务**：
    1. 下载`nssm`并解压。
    2. 在PowerShell中，进入`nssm`所在目录，运行 `./nssm install SSH_Tunnel`。
    3. 在弹出的GUI中：
       - `Path`: 填入 `C:\Windows\System32\OpenSSH\ssh.exe`。
       - `Startup directory`: 填入 `C:\Windows\System32\OpenSSH`。
       - `Arguments`: 填入 `-N -R 33890:localhost:3389 -o "ServerAliveInterval=60" -o "ExitOnForwardFailure=yes" ubuntu@remote.yourdomain.com -i C:\Users\YourUser\.ssh\id_rsa` (请将路径替换为你的私钥路径)。
       - 在 `Details` 标签页，可以设置服务的显示名称。
       - 在 `Shutdown` 标签页，勾选所有超时选项并设置为10000ms。
       - 点击 `Install service`。
    4. 启动服务：`nssm start SSH_Tunnel`。

#### 4.6.2 针对 macOS 系统

- **启用屏幕共享 (VNC)**：
  - 打开“系统设置” -> “通用” -> “共享”。
  - 打开“屏幕共享”开关。
  - 在“屏幕共享”旁边的(i)信息按钮中，可以设置访问权限和密码。VNC的默认端口是5900。

- **建立SSH反向隧道**：
  - 打开“终端” (Terminal) 应用。
  - 执行以下命令。注意端口号是5900。
    ```bash
    ssh -N -R 59000:localhost:5900 ubuntu@remote.yourdomain.com
    ```
    我们选择云服务器上的59000端口来对应macOS的5900。

- **使隧道持久化（推荐）**：
  使用macOS的`launchd`来实现开机自启动。
  - **配置SSH密钥免密登录**（同Windows部分）。
  - 创建一个`.plist`文件：
    ```bash
    nano ~/Library/LaunchAgents/com.user.sshtunnel.plist
    ```
  - 粘贴以下内容，并修改`YOUR_USERNAME`、`ubuntu`、`remote.yourdomain.com` 和私钥路径。
    ```xml
    <?xml version="1.0" encoding="UTF-8"?>
    <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
    <plist version="1.0">
    <dict>
        <key>Label</key>
        <string>com.user.sshtunnel</string>
        <key>ProgramArguments</key>
        <array>
            <string>/usr/bin/ssh</string>
            <string>-N</string>
            <string>-R</string>
            <string>59000:localhost:5900</string>
            <string>-o</string>
            <string>ServerAliveInterval=60</string>
            <string>-o</string>
            <string>ExitOnForwardFailure=yes</string>
            <string>ubuntu@remote.yourdomain.com</string>
            <string>-i</string>
            <string>/Users/YOUR_USERNAME/.ssh/id_rsa</string>
        </array>
        <key>KeepAlive</key>
        <true/>
        <key>RunAtLoad</key>
        <true/>
    </dict>
    </plist>
    ```
  - **加载服务**：
    ```bash
    launchctl load ~/Library/LaunchAgents/com.user.sshtunnel.plist
    ```

#### 4.6.3 针对 Linux 系统

- **安装并配置VNC服务器 (以Arch hyprland为例)**：
  - 安装wayvnc：`sudo pacman -S wayvnc`
  - 运行：
    ```bash
    wayvnc
    ```
    ps：密码可以留空，因为数据已经被加密。
  - VNC通常运行在5901端口 (display :1)。

- **建立SSH反向隧道**：
  ```bash
  ssh -N -R 59010:localhost:5901 ubuntu@remote.yourdomain.com
  ```

- **使隧道持久化（推荐）**：
  使用systemd服务。
  - **配置SSH密钥免密登录**。
  - 创建systemd服务文件：
    ```bash
    sudo nano /etc/systemd/system/sshtunnel.service
    ```
  - 粘贴内容，修改用户名、域名等。
    ```ini
    [Unit]
    Description=SSH Reverse Tunnel to Cloud Server
    After=network-online.target

    [Service]
    User=your_linux_username
    ExecStart=/usr/bin/ssh -N -R 59010:localhost:5901 -o "ServerAliveInterval=60" -o "ExitOnForwardFailure=yes" ubuntu@remote.yourdomain.com -i /home/your_linux_username/.ssh/id_rsa
    Restart=always
    RestartSec=3

    [Install]
    WantedBy=multi-user.target
    ```
  - **启用并启动服务**：
    ```bash
    sudo systemctl enable sshtunnel.service
    sudo systemctl start sshtunnel.service
    ```

### 4.6.4 阶段四：在 Guacamole 中配置连接

现在回到云服务器上的Guacamole Web界面 (`https://remote.yourdomain.com`)。

1. 登录 Guacamole。
2. 进入 **Settings** -> **Connections**，点击 **New Connection**。
3. 填写连接信息：

   - **对于 Windows RDP 连接**：
     - **Name**: 给连接起个名字，如 "My Windows PC"。
     - **Protocol**: 选择 `RDP`。
     - **Parameters -> Network**:
       - **Hostname**: `127.0.0.1` (或者`localhost`，因为guacd是从云服务器自身连接隧道端口)
       - **Port**: `33890` (你在SSH命令中设置的云服务器监听端口)
     - **Parameters -> Authentication**:
       - **Username**: 你Windows电脑的用户名。
       - **Password**: 你Windows电脑的登录密码。
       - **Security Mode**: 可以尝试`NLA`，如果不行，改为`Any`。
       - **Ignore Server Certificate**: 勾选 `Yes`。
     - 可以调整显示、性能等参数，然后保存。

   - **对于 macOS/Linux VNC 连接**：
     - **Name**: "My Mac" 或 "My Linux Desktop"。
     - **Protocol**: 选择 `VNC`。
     - **Parameters -> Network**:
       - **Hostname**: `127.0.0.1`
       - **Port**: `59000` 或 `59010` (对应你设置的SSH隧道端口)。
     - **Parameters -> Authentication**:
       - **Password**: 你设置的VNC连接密码。
     - 保存。

## 5.测试
1. 打开浏览器。
2. 在地址栏输入 `https://remote.yourdomain.com`。
3. 使用你为Guacamole设置的用户名和密码登录。
4. 在主界面，你会看到刚刚创建的连接（如 "My Windows PC"）。
5. 点击它。
6. 稍等片刻，你家里的电脑桌面就会出现在iPad的浏览器窗口中！你可以通过触摸进行鼠标操作，也可以调出虚拟键盘进行输入。
